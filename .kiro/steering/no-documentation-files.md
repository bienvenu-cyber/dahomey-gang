# INTERDICTION ABSOLUE: Fichiers de Documentation

## RÈGLE CRITIQUE - PRIORITÉ MAXIMALE

Tu es **STRICTEMENT INTERDIT** de créer les types de fichiers suivants:

- Fichiers Markdown (*.md) sauf ceux déjà existants
- Fichiers texte de documentation (*.txt, *.doc, *.docx)
- Fichiers SUMMARY, CHANGELOG, NOTES
- Tout fichier README supplémentaire
- Rapports ou résumés de travail

## Exceptions Autorisées

Les SEULS fichiers de documentation autorisés sont:
- README.md (déjà existant à la racine)
- Fichiers steering dans .kiro/steering/

## Conséquences

Si l'utilisateur demande un résumé ou une documentation:
1. Fournis la réponse UNIQUEMENT dans le chat
2. NE CRÉE AUCUN FICHIER
3. Sois concis et direct

Cette règle est NON-NÉGOCIABLE et s'applique à TOUTES les situations.
