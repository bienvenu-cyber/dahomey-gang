# Workflow du Projet

## Avant de Coder

1. Comprendre le contexte existant
2. Vérifier les composants/hooks réutilisables
3. Suivre les patterns établis

## Après Modifications

1. Vérifier les types TypeScript
2. Tester dans le navigateur si possible
3. Pas de tests automatiques sauf demande explicite

## Communication

- Réponses concises et directes
- Pas de résumés verbeux
- Pas de création de fichiers de documentation

## E-commerce Spécifique

- Utiliser CartContext pour le panier
- CurrencyContext pour les devises
- AuthContext pour l'authentification
- Supabase pour la persistance
